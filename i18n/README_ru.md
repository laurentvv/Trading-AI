## 🌟 О проекте

Этот проект представляет собой экспертную систему поддержки принятия решений для торговли ETF, использующую тримодальный гибридный подход ИИ. Он разработан для обеспечения всестороннего и надежного анализа путем объединения нескольких перспектив ИИ.

### 🚀 Стратегия двойного тикера (Анализ vs Торговля)
Система использует инновационный подход для максимизации точности модели:
- **Высокоточный анализ**: Модели ИИ анализируют **мировые эталонные индексы** (`^NDX` для Nasdaq, `CL=F` для нефти WTI). Эти индексы предлагают более длительную историю и «чистые» тренды, без шума, связанного с часами торговли или комиссиями ETF.
- **Исполнение через ETF**: Реальные ордера размещаются на соответствующих тикерах в **Trading 212** (`SXRV.DE`, `CRUDP.PA`), используя **актуальные цены T212** (через API позиций) для определения объема позиции.

### 🧠 Гибридный движок ИИ
Система объединяет восемь различных сигналов:
1.  **Классическая количественная модель**: Ансамбль RandomForest/GradientBoosting/LogisticRegression, обученный на технических и макроэкономических индикаторах.
2.  **TimesFM 2.5 (Google Research)**: Современная базовая модель для прогнозирования временных рядов.
3.  **Модель Oil-Bench (Gemma 4:e4b)**: Специализированная энергетическая модель, объединяющая фундаментальные данные **EIA** (запасы, импорт, использование НПЗ) и настроения для торговли WTI.
4.  **Текстовая LLM (Gemma 4:e4b)**: Контекстуальный анализ необработанных данных, новости в реальном времени через навык **AlphaEar** и интеграция динамических **макроэкономических веб-исследований**.
5.  **Визуальная LLM (Gemma 4:e4b)**: Прямой анализ технических графиков (`enhanced_trading_chart.png`).
6.  **Анализ настроений**: Гибридный анализ, сочетающий Alpha Vantage и «горячие» тренды из **AlphaEar** (Weibo, WallstreetCN).
7.  **Децентрализованные данные (Hyperliquid)**: Анализ спекулятивных настроений по нефти (WTI) через *ставку финансирования* и *открытый интерес*.
8.  **Модель Венсана Ганна**: Геополитический и кросс-активный анализ (WTI, Brent, Gas, DXY, MA200) для обнаружения макроэкономического дна.

Цель состоит в том, чтобы выработать окончательное решение (`BUY`, `SELL`, `HOLD`) с абсолютным приоритетом на **точность прежде всего**.

### 🧘 Философия принятия решений: «Когнитивная осмотрительность»
В отличие от классических торговых алгоритмов, которые впадают в панику при резком росте волатильности, эта система применяет подход информированного инвестора:
- **Требуется сильный консенсус**: Количественная модель (классическая) может сигнализировать о продаже (`SELL`), но если когнитивные модели (текстовая LLM, зрение, TimesFM) остаются нейтральными, система предпочтет `HOLD`.
- **Фильтр уверенности**: Решение о движении (покупка или продажа) подтверждается только в том случае, если общая уверенность превышает порог безопасности (обычно 40%). Ниже этого уровня система рассматривает сигнал как «шум» и остается в режиме ожидания.
- **Защита капитала**: В режиме `VERY_HIGH` риска `HOLD` служит щитом. Он предотвращает вход на нестабильный рынок и позволяет избежать преждевременного выхода при простой технической коррекции, если фундаментальные факторы (новости/зрение/Hyperliquid) не подтверждают неизбежный крах.

### ✨ Ключевые особенности

- **Подход двойного тикера**: Анализ индекса, торговля ETF.
- **Актуальные цены T212**: Получение цен в EUR в реальном времени через API Trading 212 (0.2 сек) с резервным использованием yfinance и кэшированием в parquet.
- **Сетевая устойчивость**: Автоматический выключатель yfinance с отдельными трекерами (информация vs загрузка), тайм-аут 10 сек для всех сетевых вызовов.
- **Продвинутое познание**: Использование **Gemma 4** для лучшего технического/фундаментального синтеза.
- **Новости и настроения блокчейна**: Интеграция **AlphaEar** и **Hyperliquid** для отслеживания социальных и спекулятивных настроений.
- **Автоматизированный планировщик**: Скрипт `schedule.py` для непрерывного выполнения (8:30 – 18:00) на сервере.
- **Продвинутое управление рисками**: Автоматическая корректировка сигналов в зависимости от волатильности и рыночного режима.

### 💻 Технологический стек

- **Язык**: `Python 3.12+`
- **Вычисления и данные**: `pandas`, `numpy`, `yfinance`, `pyarrow`, `pandas_datareader`, `hyperliquid-python-sdk`
- **Машинное обучение**: `scikit-learn`, `shap`
- **ИИ и LLM**: `requests`, `ollama`
- **Веб-скрейпинг и поиск**: `beautifulsoup4`, `duckduckgo_search`, `crawl4ai`
- **Визуализация**: `matplotlib`, `seaborn`, `mplfinance`
- **Утилиты**: `tqdm`, `rich`, `python-dotenv`, `schedule`

### ⚙️ Производительность и оборудование
Система разработана для **эффективной работы на обычном оборудовании** без необходимости в выделенном GPU.
- **Только CPU**: Инференс LLM (Gemma 4 через Ollama) и TimesFM оптимизированы для быстрого выполнения на CPU при наличии достаточного объема оперативной памяти.
- **Рекомендуемая память**: минимум 16 ГБ (32 ГБ рекомендуется для комфортной работы Gemma 4).
- **Время выполнения**: ~2–5 минут на полный цикл (включая веб-сканирование, обучение ML, прогнозы TimesFM и 3 анализа LLM).
- **Скорость API**: сверхбыстрая интеграция с Trading 212 (менее 1 сек для получения актуальной цены).

## 📂 Структура проекта

Проект организован модульно для удобства обслуживания.

```
Trading-AI/
├── src/                     # Основные модули
│   ├── eia_client.py               # Клиент фундаментальных данных по энергетике
│   ├── oil_bench_model.py          # Специализированная модель по энергетике
│   ├── enhanced_decision_engine.py # Движок слияния и модель Венсана Ганна
│   ├── advanced_risk_manager.py    # Управление рисками с учетом тренда
│   ├── adaptive_weight_manager.py  # Динамическое взвешивание моделей
│   ├── t212_executor.py            # Реальное исполнение на Trading 212
│   ├── timesfm_model.py            # Интеграция TimesFM 2.5
│   └── ...                         # Данные, признаки, клиент LLM
├── tests/                   # Скрипты тестирования и валидации
├── data_cache/              # Рыночные и макроданные (Parquet)
├── main.py                  # Единая точка входа (анализ и торговля)
├── schedule.py              # Живой планировщик (8:30 – 18:00)
├── backtest_engine.py       # Движок исторического бэктестинга
├── .env                     # API-ключи (Alpha Vantage, T212, EIA)
└── README.md                # Эта документация
```

## 🚀 Быстрый старт

Следуйте этим шагам, чтобы настроить локальную среду разработки.

### ✅ Требования

- Python 3.12+ (через `uv`)
- Установленный и запущенный локально [Ollama](https://ollama.com/).
- Загруженная модель LLM: `ollama pull gemma4:e4b`

### ⚙️ Установка

1.  **Клонируйте репозиторий:**
    ```sh
    git clone https://github.com/laurentvv/Trading-AI.git
    cd Trading-AI
    ```
2.  **Установите `uv` (если еще не установлено):**
    Инструкции по установке см. на [astral.sh/uv](https://astral.sh/uv).

3.  **Установите и пропатчите TimesFM 2.5 (ВАЖНЫЙ шаг):**
    Запустите скрипт установки, чтобы клонировать модель в `vendor/` и применить патчи:
    ```bash
    python setup_timesfm.py
    ```

4.  **Инициализируйте и синхронизируйте среду:**
    ```bash
    uv sync
    ```

5.  **Установите браузеры для веб-исследований (Crawl4AI):**
    ```bash
    uv run python -m playwright install chromium
    ```

6.  **Настройте API-ключи:**
    Создайте файл `.env` в корне проекта:
    ```
    ALPHA_VANTAGE_API_KEY="ВАШ_КЛЮЧ"
    EIA_API_KEY="ВАШ_КЛЮЧ"
    ```

## 🛠️ Использование

Система обучает свои модели на самых свежих данных при каждом запуске перед принятием решения.

### Режим симуляции (Paper Trading)

Чтобы протестировать систему без риска с фиктивным капиталом в 1000 евро, используйте флаг `--simul`. Система будет вести строгую историю покупок и продаж.

```sh
# Запустить симуляционный анализ (по умолчанию: SXRV.DE - Nasdaq 100 EUR)
uv run main.py --simul

# Запустить для нефти (WTI)
uv run main.py --ticker CRUDP.PA --simul
```

### Реальное исполнение (Trading 212)

Система теперь **полностью интегрирована** с Trading 212:
- **Проверка портфеля**: Перед любым действием робот проверяет ваши реальные денежные средства и позиции.
- **Управление API**: Включает механизмы автоматического повтора при достижении лимитов запросов (Rate Limiting).

```sh
# Запустить анализ с реальным исполнением (Demo или Real в зависимости от .env)
uv run main.py --t212
```
