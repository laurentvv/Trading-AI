## 🌟 حول المشروع

هذا المشروع هو نظام خبير لدعم القرار في تداول صناديق الاستثمار المتداولة (ETFs)، يستخدم نهج الذكاء الاصطناعي الهجين ثلاثي الأنماط. تم تصميمه لتوفير تحليل شامل وقوي من خلال الجمع بين عدة وجهات نظر للذكاء الاصطناعي.

### 🚀 استراتيجية الرمز المزدوج (التحليل مقابل التداول)
يستخدم النظام نهجاً مبتكراً لزيادة دقة النموذج:
- **تحليل عالي الدقة**: تقوم نماذج الذكاء الاصطناعي بتحليل **المؤشرات المرجعية العالمية** (`^NDX` لناسداك، و `CL=F` لنفط خام غرب تكساس الوسيط). تقدم هذه المؤشرات تاريخاً أطول واتجاهات "أنقى"، دون الضجيج المرتبط بساعات التداول أو رسوم صناديق الاستثمار المتداولة.
- **تنفيذ صناديق الاستثمار المتداولة**: يتم وضع الأوامر الحقيقية على الرموز المقابلة في **Trading 212** (`SXRV.DE`, `CRUDP.PA`) باستخدام **أسعار T212 المباشرة** (عبر واجهة برمجة تطبيقات الصفقات) لتحديد حجم الصفقة.

### 🧠 محرك الذكاء الاصطناعي الهجين
يدمج النظام ثماني إشارات متميزة:
1.  **النموذج الكمي الكلاسيكي**: مجموعة من RandomForest/GradientBoosting/LogisticRegression مدربة على المؤشرات الفنية والاقتصادية الكلية.
2.  **TimesFM 2.5 (Google Research)**: نموذج أساسي متطور للتنبؤ بالسلاسل الزمنية.
3.  **نموذج Oil-Bench (Gemma 4:e4b)**: نموذج متخصص في الطاقة يدمج البيانات الأساسية من **EIA** (المخزونات، الواردات، استهلاك المصافي) والمشاعر لتداول النفط (WTI).
4.  **نموذج اللغة النصي (Gemma 4:e4b)**: تحليل سياقي للبيانات الخام، والأخبار في الوقت الفعلي عبر مهارة **AlphaEar**، ودمج **أبحاث الويب الاقتصادية الكلية** الديناميكية.
5.  **نموذج اللغة المرئي (Gemma 4:e4b)**: تحليل مباشر للرسوم البيانية الفنية (`enhanced_trading_chart.png`).
6.  **تحليل المشاعر**: تحليل هجين يجمع بين Alpha Vantage والاتجاهات "الساخنة" من **AlphaEar** (Weibo, WallstreetCN).
7.  **البيانات اللامركزية (Hyperliquid)**: تحليل مشاعر المضاربة على النفط (WTI) عبر *معدل التمويل* و *الفائدة المفتوحة*.
8.  **نموذج فنسنت غان**: تحليل جيوسياسي وعبر الأصول (WTI, Brent, Gas, DXY, MA200) للكشف عن القيعان الاقتصادية الكلية.

الهدف هو إنتاج قرار نهائي (`BUY`, `SELL`, `HOLD`) مع أولوية مطلقة لـ **الدقة أولاً**.

### 🧘 فلسفة القرار: "الحذر المعرفي"
على عكس خوارزميات التداول الكلاسيكية التي تصاب بالذعر بمجرد انفجار التقلبات، يطبق هذا النظام نهج المستثمر المطلع:
- **الإجماع القوي مطلوب**: قد يطلق النموذج الكمي (الكلاسيكي) إنذاراً (`SELL`)، ولكن إذا ظلت النماذج المعرفية (النصية، المرئية، TimesFM) محايدة، سيفضل النظام خيار `HOLD`.
- **فلتر الثقة**: لا يتم التحقق من صحة قرار الحركة (شراء أو بيع) إلا إذا تجاوزت الثقة العالمية حد الأمان (عادةً 40%). أقل من ذلك، يعتبر النظام الإشارة "ضجيجاً" ويبقى في وضع الاستعداد.
- **حماية رأس المال**: في وضع المخاطرة `VERY_HIGH` ، يعمل خيار `HOLD` كدرع. فهو يمنع دخول سوق غير مستقر ويتجنب الخروج المبكر عند تصحيح فني بسيط إذا لم تؤكد الأساسيات (الأخبار/الرؤية/Hyperliquid) حدوث انهيار وشيك.

### ✨ الميزات الرئيسية

- **نهج الرمز المزدوج**: تحليل المؤشر، وتداول صندوق الاستثمار المتداول.
- **أسعار T212 المباشرة**: استعادة أسعار اليورو في الوقت الفعلي عبر واجهة برمجة تطبيقات Trading 212 (0.2 ثانية)، مع توفر yfinance كبديل وذاكرة تخزين مؤقت بتنسيق parquet.
- **مرونة الشبكة**: قاطع دائرة yfinance مع متتبعات منفصلة (معلومات مقابل تنزيل)، ومهلة 10 ثوانٍ على جميع استدعاءات الشبكة.
- **الإدراك المتقدم**: استخدام **Gemma 4** لتحسين التوليف الفني/الأساسي.
- **أخبار ومشاعر البلوكشين**: دمج **AlphaEar** و **Hyperliquid** لالتقاط المشاعر الاجتماعية والمضاربة.
- **مجدول آلي**: نص `schedule.py` للتنفيذ المستمر (8:30 صباحاً - 6:00 مساءً) على خادم.
- **إدارة مخاطر متقدمة**: تعديل تلقائي للإشارة بناءً على التقلبات ونظام السوق.

### 💻 التقنيات المستخدمة

- **اللغة**: `Python 3.12+`
- **الحسابات والبيانات**: `pandas`, `numpy`, `yfinance`, `pyarrow`, `pandas_datareader`, `hyperliquid-python-sdk`
- **تعلم الآلة**: `scikit-learn`, `shap`
- **الذكاء الاصطناعي ونماذج اللغة**: `requests`, `ollama`
- **البحث وكشط الويب**: `beautifulsoup4`, `duckduckgo_search`, `crawl4ai`
- **التصور**: `matplotlib`, `seaborn`, `mplfinance`
- **الأدوات المساعدة**: `tqdm`, `rich`, `python-dotenv`, `schedule`

### ⚙️ الأداء والأجهزة
تم تصميم النظام ليكون **فعالاً على أجهزة المستهلك العادية** دون الحاجة إلى وحدة معالجة رسومات (GPU) مخصصة.
- **المعالج فقط**: تم تحسين استنتاج نماذج اللغة (Gemma 4 عبر Ollama) و TimesFM للتنفيذ السريع على المعالج إذا توفرت ذاكرة وصول عشوائي كافية.
- **الذاكرة الموصى بها**: 16 جيجابايت كحد أدنى (يُقترح 32 جيجابايت لتشغيل Gemma 4 بشكل مريح).
- **وقت التنفيذ**: حوالي 2 إلى 5 دقائق لدورة كاملة (بما في ذلك زحف الويب، تدريب تعلم الآلة، تنبؤات TimesFM، و3 تحليلات لنماذج اللغة).
- **سرعة الواجهة البرمجية**: تكامل فائق السرعة مع Trading 212 (أقل من ثانية لاستعادة السعر المباشر).

## 🚀 البدء السريع

اتبع هذه الخطوات لإعداد بيئة التطوير المحلية الخاصة بك.

### ✅ المتطلبات الأساسية

- بايثون 3.12+ (عبر `uv`)
- تثبيت [Ollama](https://ollama.com/) وتشغيله محلياً.
- تحميل نموذج اللغة: `ollama pull gemma4:e4b`

### ⚙️ التثبيت

1.  **استنساخ المستودع:**
    ```sh
    git clone https://github.com/laurentvv/Trading-AI.git
    cd Trading-AI
    ```
2.  **تثبيت `uv` (إذا لم يتم تثبيته بعد):**
    راجع [astral.sh/uv](https://astral.sh/uv) للحصول على تعليمات التثبيت.

3.  **تثبيت وإصلاح TimesFM 2.5 (خطوة حاسمة):**
    قم بتشغيل نص التثبيت لاستنساخ النموذج في `vendor/` وتطبيق الإصلاحات:
    ```bash
    python setup_timesfm.py
    ```

4.  **تهيئة ومزامنة البيئة:**
    ```bash
    uv sync
    ```

5.  **تثبيت المتصفحات لأبحاث الويب (Crawl4AI):**
    ```bash
    uv run python -m playwright install chromium
    ```

6.  **تكوين مفاتيح الواجهة البرمجية الخاصة بك:**
    أنشئ ملف `.env` في جذر المشروع:
    ```
    ALPHA_VANTAGE_API_KEY="YOUR_KEY"
    EIA_API_KEY="YOUR_KEY"
    ```

## 🛠️ الاستخدام

يقوم النظام بتدريب نماذجه على أحدث البيانات عند كل تنفيذ قبل إعطاء القرار.

### وضع المحاكاة (التداول الافتراضي)

لاختبار النظام دون مخاطرة برأس مال وهمي قدره 1000 يورو، استخدم علم `--simul`. سيقوم النظام بإدارة سجل صارم للمشتريات والمبيعات.

```sh
# تشغيل تحليل محاكى (افتراضياً: SXRV.DE - ناسداك 100 باليورو)
uv run main.py --simul

# التشغيل على النفط (WTI)
uv run main.py --ticker CRUDP.PA --simul
```

### التنفيذ الحقيقي (Trading 212)

النظام الآن **متكامل تماماً** مع Trading 212:
- **التحقق من المحفظة**: قبل أي إجراء، يستشير الروبوت نقودك وصفقاتك الحقيقية.
- **إدارة الواجهة البرمجية**: تتضمن آليات إعادة المحاولة التلقائية ضد حدود الطلبات (Rate Limiting).

```sh
# تشغيل التحليل مع التنفيذ الحقيقي (تجريبي أو حقيقي حسب ملف .env)
uv run main.py --t212
```
