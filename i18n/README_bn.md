## 🌟 প্রজেক্ট সম্পর্কে

এই প্রজেক্টটি একটি ত্রি-মোডাল হাইব্রিড এআই পদ্ধতি ব্যবহার করে ইটিএফ ট্রেডিংয়ের জন্য একটি এক্সপার্ট ডিসিশন-সাপোর্ট সিস্টেম। এটি বিভিন্ন এআই দৃষ্টিকোণ একত্রিত করে একটি ব্যাপক এবং মজবুত বিশ্লেষণ প্রদানের জন্য ডিজাইন করা হয়েছে।

### 🚀 ডুয়েল-টিকার স্ট্র্যাটেজি (অ্যানালাইসিস বনাম ট্রেডিং)
মডেলের নির্ভুলতা বাড়াতে সিস্টেমটি একটি উদ্ভাবনী পদ্ধতি ব্যবহার করে:
- **হাই-ফিডেলিটি অ্যানালাইসিস**: এআই মডেলগুলি **গ্লোবাল রেফারেন্স সূচক** (`^NDX` নাসডাকের জন্য, `CL=F` ডব্লিউটিআই ক্রুড অয়েলের জন্য) বিশ্লেষণ করে। এই সূচকগুলি ট্রেডিং আওয়ার বা ইটিএফ ফি সম্পর্কিত নয়েজ ছাড়াই দীর্ঘতর ইতিহাস এবং "খাঁটি" ট্রেন্ড সরবরাহ করে।
- **ইটিএফ এক্সিকিউশন**: রিয়েল অর্ডারগুলি **Trading 212**-এ সংশ্লিষ্ট টিকারগুলিতে (`SXRV.DE`, `CRUDP.PA`) প্লেস করা হয়, পজিশন সাইজিংয়ের জন্য **T212 লাইভ প্রাইস** (পজিশন এপিআই এর মাধ্যমে) ব্যবহার করে। পোর্টফোলিও স্টেট সরাসরি T212 থেকে সিঙ্ক করা হয় (`sync_state_from_t212()`), এবং লাইভ প্রাইস অ্যানালাইসিস পাইপলাইনে ইনজেক্ট করা হয় (`src/data.py` এ `_inject_t212_live_price()`)।

### 🧠 হাইব্রিড এআই ইঞ্জিন
সিস্টেমটি এগারোটি স্বতন্ত্র সিগন্যাল একত্রিত করে:
1.  **ক্লাসিক কোয়ান্টিটেটিভ মডেল**: টেকনিক্যাল এবং ম্যাক্রোইকোনমিক ইন্ডিকেটরের ওপর ট্রেন্ড করা RandomForest/GradientBoosting/LogisticRegression এনসেম্বল।
2.  **TimesFM 2.5 (Google Research)**: টাইম-সিরিজ ফোরকাস্টিংয়ের জন্য অত্যাধুনিক ফাউন্ডেশন মডেল।
3.  **TensorTrade / PPO (Reinforcement Learning)**: রিইনফোর্সমেন্ট লার্নিং (RL) এজেন্ট (stable-baselines3) সাইকেলগুলোর মধ্যে পারসিস্টেন্স সহ একটি কাস্টম জিমনেসিয়াম ট্রেডিং এনভায়রনমেন্টে একটি PPO পলিসি ট্রেন করে।
4.  **Oil-Bench মডেল (Gemma 4 12B (Unsloth))**: এনার্জি-স্পেশালাইজড মডেল যা WTI ট্রেডিংয়ের জন্য **EIA** ফান্ডামেন্টাল ডেটা (স্টক, ইমপোর্ট, রিফাইনারি ইউটিলাইজেশন) এবং সেন্টিনমেন্ট একত্রিত করে।
5.  **Textual LLM (Gemma 4 12B (Unsloth))**: কাঁচা ডেটার প্রাসঙ্গিক বিশ্লেষণ, **AlphaEar** স্কিলের মাধ্যমে রিয়েল-টাইম খবর, এবং গতিশীল **সামষ্টিক অর্থনৈতিক ওয়েব গবেষণার** একত্রীকরণ। সিদ্ধান্ত নেওয়ার আগে গভীর মৌলিক সচেতনতা অর্জনের জন্য এটি স্পষ্টভাবে রাতারাতি **Morning Brief** প্রতিবেদনটি গ্রহণ করে।
6.  **ভিজ্যুয়াল LLM (Gemma 4 12B (Unsloth))**: সরাসরি টেকনিক্যাল চার্ট বিশ্লেষণ (`enhanced_trading_chart.png`)।
7.  **অ্যাডভান্সড কগনিশন**: রিজনিং টোকেন `<|think|>` সমস্ত চারটি প্রোডাকশন সিস্টেম প্রম্পটে **সক্রিয়** রয়েছে। স্কিমা লেয়ারটি কঠোর স্কিমা কেসগুলি নিশ্চিত করে যা পরিষ্কার JSON তৈরি করে। বিস্তারিত তথ্যের জন্য `docs/ADR-001-think-mode-dual-layer-defence.md` দেখুন।
    - **স্বায়ত্তশাসিত মর্নিং ব্রিফ এজেন্ট**: একটি রাতারাতি `smolagents` ভিত্তিক ওয়ার্কফ্লো (`morning_brief/morning_brief.py`) যা `schedule.py` এর মাধ্যমে সকাল ১:০০ টায় স্বয়ংক্রিয়ভাবে চলে। এটি স্বাধীনভাবে দৈনিক API লগ ক্রল করে এবং মৌলিক EIA ইনভেন্টরি ডেটা ডাউনলোড করে। জেনারেট করা মার্কডাউন রিপোর্টটি (`morning_market_brief.md`) দৈনিক ট্রেডিং চক্রের সময় স্বয়ংক্রিয়ভাবে টেক্সচুয়াল LLM-এর সিস্টেম প্রম্পটে ইনজেক্ট করা হয়।
    - **সংবাদ এবং ব্লকচেইন সেন্টিমেন্ট**: সামাজিক এবং অনুমানমূলক সেন্টিমেন্ট ক্যাপচার করতে **AlphaEar** এবং **Hyperliquid**-এর ইন্টিগ্রেশন।
8.  **ডিসেন্ট্রালাইজড ডেটা (Hyperliquid)**: *Funding Rate* এবং *Open Interest* এর মাধ্যমে ওয়েল (WTI) এ স্পেকুলেটিভ সেন্টিমেন্ট অ্যানালাইসিস।
9.  **Vincent Ganne মডেল**: ম্যাক্রোইকোনমিক বটম শনাক্তকরণের জন্য জিওপলিটিক্যাল এবং ক্রস-অ্যাসেট বিশ্লেষণ (WTI, Brent, Gas, DXY, MA200)।
10. **Grebenkov মডেল**: অ্যাগনস্টিক রিস্ক প্যারিটি (Agnostic Risk Parity) ব্যবহার করে ক্রস-অ্যাসেট বিশ্লেষণের জন্য ক্যালিব্রেট করা ট্রেন্ড-ফলোয়িং ম্যাথমেটিক্যাল মডেল।
11. **হাইব্রিড ফিউশন ইঞ্জিন**: মেটা-মডেল যা সমস্ত সাব-মডেল জুড়ে ডায়নামিক ওয়েটিং এবং কগনিটিভ কনসেনসাস অর্কেস্ট্রেট করে।

এর লক্ষ্য হলো চূড়ান্ত সিদ্ধান্ত (`BUY`, `SELL`, `HOLD`) তৈরি করা যেখানে **নির্ভুলতার** উপর পরম অগ্রাধিকার দেওয়া হয়।

### 🧘 ডিসিশন ফিলোসফি: "কগনিটিভ প্রুডেন্স (Cognitive Prudence)"
ক্লাসিক ট্রেডিং অ্যালগরিদমগুলোর বিপরীতে, যা ভোলাটিলিটি বাড়লেই প্যানিক করে, এই সিস্টেমটি একজন সচেতন বিনিয়োগকারীর পদ্ধতি প্রয়োগ করে:
- **স্ট্রং কনসেনসাস রিকোয়ার্ড**: একটি কোয়ান্টিটেটিভ মডেল (ক্লাসিক) হয়তো সংকেত দিতে পারে (`SELL`), তবে যদি কগনিটিভ মডেলগুলো (Text LLM, Vision, TimesFM) নিরপেক্ষ থাকে, তবে সিস্টেমটি `HOLD` পছন্দ করবে।
- **কনফিডেন্স ফিল্টার**: একটি মুভমেন্ট সিদ্ধান্ত (Buy বা Sell) তখনই ভ্যালিডেট হয় যদি সামগ্রিক কনফিডেন্স একটি সেফটি থ্রেশহোল্ড (সাধারণত ৪০%) অতিক্রম করে। এর নিচে, সিস্টেমটি সিগন্যালটিকে "নয়েজ" হিসেবে বিবেচনা করে এবং স্ট্যান্ডবাই থাকে।
- **ক্যাপিটাল প্রোটেকশন**: `VERY_HIGH` রিস্ক মোডে, `HOLD` একটি শিল্ড হিসেবে কাজ করে। এটি একটি অস্থিতিশীল মার্কেটে প্রবেশ করতে বাধা দেয় এবং যদি ফান্ডামেন্টাল (News/Vision/Hyperliquid) কোনো আসন্ন ক্র্যাশ নিশ্চিত না করে তবে সাধারণ টেকনিক্যাল কারেকশনে অকালে বেরিয়ে যাওয়া এড়ায়।

### ✨ মূল বৈশিষ্ট্যসমূহ

- **ডুয়েল-টিকার অ্যাপ্রোচ**: সূচক বিশ্লেষণ করুন, ইটিএফ ট্রেড করুন।
- **T212 লাইভ প্রাইস**: Trading 212 API (0.2s) এর মাধ্যমে ইউরো প্রাইস রিয়েল-টাইমে রিকভারি, ಜೊತೆಗೆ yfinance ফলব্যাক এবং পারকেট (parquet) ক্যাশ।
- **ডেটেড ব্রেন্ট স্প্রেড**: ব্রেন্ট স্পট (ডেটেড) এবং ব্রেন্ট ফিউচারের মধ্যকার স্প্রেডের মাধ্যমে ফিজিক্যাল মার্কেটের টেনশন পর্যবেক্ষণ।
- **নেটওয়ার্ক রেসিলিয়েন্স**: আলাদা ট্র্যাকার (তথ্য বনাম ডাউনলোড) সহ yfinance সার্কিট ব্রেকার, সমস্ত নেটওয়ার্ক কলে ১০ সেকেন্ডের টাইমআউট।
- **ক্যাশ অটো-ইনভ্যালিডেশন**: পারকেট ক্যাশ স্বয়ংক্রিয়ভাবে স্টেলনেস (> ২ দিন) সনাক্ত করে এবং রিফ্রেশ করতে বাধ্য করে। ম্যানুয়ালি ক্যাশ পরিষ্কার করতে `refresh_cache.py` ব্যবহার করুন।
- **LLM কল প্যারালেলাইজেশন**: ইনডিপেনডেন্ট মডেল কলগুলি (`text_llm`, `visual_llm`, `search_query`, `timesfm`, `tensortrade`, `grebenkov`) I/O এর সাথে Ollama ইনফারেন্সকে ওভারল্যাপ করতে একটি `ThreadPoolExecutor`-এ রান করে। CPU-তে সাধারণত ৪-৬ মিনিট সময় লাগে, যেখানে সিক্যুয়েনশিয়ালে ১০+ মিনিট লাগে।
- **24h সার্চ-কোয়েরি ক্যাশ**: LLM-জেনারেটেড ওয়েব-সার্চ কোয়েরি `data_cache/search_queries/<ticker>_<date>_<price-sig>.json` এর অধীনে ক্যাশ করা হয়। তারিখ + একটি প্রাইস-অ্যাকশন সিগনেচার দিয়ে কি করা হয়, তাই একটি রেজিম পরিবর্তনে এটি বাতিল হয়ে যায়। ফলব্যাক কোয়েরিগুলি **কখনোই** ক্যাশ করা হয় না।
- **হার্ড সাইকেল টাইমআউট**: প্রতিটি টিকার সাইকেল একটি ১৫ মিনিটের বাজেটে মোড়ানো থাকে (`main.py` এ `CYCLE_TIMEOUT_SECONDS`)। টাইমআউটে, ওয়ার্কার থ্রেডকে `shutdown(wait=False)` করা হয় যাতে পরবর্তী টিকার অবিলম্বে শুরু হয়; টাইম-আউট হওয়া টিকারে HOLD প্রয়োগ করা হয়।
- **অরফান-থ্রেড সেফটি**: সাইকেল টাইমআউট হলে, প্রতিটি টিকারের জন্য একটি `threading.Event` সেট করা হয় যাতে কোনো `execute_t212_trade` কলের আগেই অরফান ওয়ার্কারটি বের হয়ে যায় — ব্যবহারকারীকে "HOLD appliqué" প্যানেল দেখানোর পরে রিয়েল-মানি ট্রেডগুলি প্রতিরোধ করতে।
- **LLM ফেইলিওর সেন্টিনেল**: যখন `_query_ollama` সমস্ত রিট্রাই শেষ করে, ফলব্যাক ডিক্ট একটি `"failed": True` ফ্ল্যাগ বহন করে।
- **অ্যাডভান্সড কগনিশন**: **Gemma 4 12B** এবং **ডুয়েল-লেয়ার JSON ডিফেন্স** ব্যবহার:
  1. **সার্ভার-সাইড স্কিমা এনফোর্সমেন্ট** (`format: SCHEMA_*` এবং `additionalProperties: false`)।
  2. **ডিফেন্সিভ সিস্টেম-প্রম্পট সাফিক্স** (`"...never add a 'thought' key."`)।

  `<|think|>` রিজনিং টোকেনটি সমস্ত ৪টি প্রোডাকশন সিস্টেম প্রম্পটে **সক্রিয়**।
- **নিউজ ও ব্লকচেইন সেন্টিমেন্ট**: সামাজিক ও স্পেকুলেটিভ সেন্টিমেন্ট ধরতে **AlphaEar** এবং **Hyperliquid** এর ইন্টিগ্রেশন।
- **অটোমেটেড শিডিউলার**: সার্ভারে ক্রমাগত এক্সিকিউশনের (সকাল ৮:৩০ - সন্ধ্যা ৬:০০) জন্য `schedule.py` স্ক্রিপ্ট।
- **সেন্ট্রালাইজড রিস্ক ম্যানেজমেন্ট**: `AdvancedRiskManager` অ্যান্টি-লস (Stop-Loss) এবং ট্রেইলিং স্টপ লজিক সেন্ট্রালাইজ করে।
- **স্ট্রিক্ট ডেটা কন্ট্রাক্টস**: সমস্ত এআই মডেল সম্পূর্ণরূপে একটি স্ট্রংলি-টাইপড `ModelResult` ডেটাক্লাস (`signal`, `confidence`, `reasoning`) রিটার্ন করার জন্য স্ট্যান্ডার্ডাইজ করা হয়েছে।
- **কোড হেলথ অডিটেড**: স্বয়ংক্রিয় অডিটের মাধ্যমে প্রজেক্টটি একটি **Grade B** কোড হেলথ স্ট্যান্ডার্ড বজায় রাখে।
- **প্রোডাকশন ব্যাকটেস্টিং**: রিয়েল প্রাইস এবং T212 ফি দিয়ে রিয়েল প্রোড সিগন্যালগুলো রিপ্লে করার জন্য স্ট্যান্ডঅ্যালোন ব্যাকটেস্ট ইঞ্জিন (`backtest_prod.py`)।
- **ডিবাগ ডাম্প কন্ট্রোল**: ক্যাপড (৫ মেগাবাইট) `data_cache/llm_debug_fail.txt` LLM-ফেইলিওর ডাম্প নিষ্ক্রিয় করতে `TRADING_DEBUG_DUMP=0` সেট করুন।

## 🚀 কুইক স্টার্ট

### ⚙️ ইনস্টলেশন

1.  **রিপোজিটরি ক্লোন করুন:**
    ```sh
    git clone https://github.com/laurentvv/Trading-AI.git
    cd Trading-AI
    ```
2.  **`uv` ইনস্টল করুন (যদি আগে না করা থাকে):**
    ইনস্টলেশন নির্দেশাবলীর জন্য [astral.sh/uv](https://astral.sh/uv) দেখুন।

3.  **ভার্চুয়াল এনভায়রনমেন্ট তৈরি ও সক্রিয় করুন (অত্যন্ত গুরুত্বপূর্ণ):**
    ফাউন্ডেশন মডেলগুলি ইনস্টল করার আগে আপনাকে অবশ্যই `.venv` তৈরি এবং সক্রিয় করতে হবে।
    ```bash
    uv venv
    source .venv/bin/activate  # উইন্ডোজে `.\.venv\Scripts\activate.ps1` ব্যবহার করুন
    ```

4.  **ফাউন্ডেশন মডেল ইনস্টল করুন:**
    মডেলগুলোকে `vendor/` এ ক্লোন করতে এবং প্যাচ প্রয়োগ করতে ইনস্টলেশন স্ক্রিপ্ট চালান:
    ```bash
    python setup_timesfm.py
    ```

5.  **এনভায়রনমেন্ট ইনিশিয়ালাইজ ও সিঙ্ক্রোনাইজ করুন:**
    ```bash
    uv sync
    ```

6.  **ওয়েব রিসার্চের (Crawl4AI) জন্য ব্রাউজার ইনস্টল করুন:**
    ```bash
    uv run python -m playwright install chromium
    ```

7.  **আপনার API কীগুলি কনফিগার করুন:**
    প্রজেক্ট রুটে একটি `.env` ফাইল তৈরি করুন:
    ```env
    ALPHA_VANTAGE_API_KEY="YOUR_KEY"
    EIA_API_KEY="YOUR_KEY"
    ```

## 🛠️ ব্যবহার

সিদ্ধান্ত দেওয়ার আগে সিস্টেমটি প্রতিটি এক্সিকিউশনে সর্বশেষ ডেটার উপর তার মডেলগুলিকে ট্রেন করে।

### সিমুলেশন মোড (পেপার ট্রেডিং)

€১০০০ এর একটি কাল্পনিক মূলধন দিয়ে ঝুঁকি ছাড়াই সিস্টেমটি টেস্ট করতে `--simul` ফ্ল্যাগ ব্যবহার করুন। সিস্টেমটি ক্রয় ও বিক্রয়ের একটি কঠোর ইতিহাস পরিচালনা করবে।

```sh
# একটি সিমুলেটেড অ্যানালাইসিস চালান (ডিফল্ট: SXRV.DE - Nasdaq 100 EUR)
uv run main.py --simul

# ওয়েল (WTI) এ রান করুন
uv run main.py --ticker CRUDP.PA --simul
```

### রিয়েল এক্সিকিউশন (Trading 212)

সিস্টেমটি এখন সম্পূর্ণরূপে Trading 212-এর সাথে **ইন্টিগ্রেটেড**:
- **পোর্টফোলিও ভেরিফিকেশন**: যেকোনো পদক্ষেপের আগে, রোবটটি আপনার রিয়েল ক্যাশ এবং পজিশনগুলি কনসাল্ট করে।
- **API ম্যানেজমেন্ট**: রিকোয়েস্ট লিমিটের (Rate Limiting) বিপরীতে স্বয়ংক্রিয় রিট্রাই মেকানিজম অন্তর্ভুক্ত।

```sh
# রিয়েল এক্সিকিউশনের সাথে অ্যানালাইসিস চালান (.env অনুযায়ী ডেমো বা রিয়েল)
uv run main.py --t212
```

## 🧪 প্রোডাকশন ব্যাকটেস্টিং

### বৈশিষ্ট্যসমূহ
- **রিয়েল সিগন্যাল**: 11-মডেল হাইব্রিড ইঞ্জিনের সঠিক সিদ্ধান্তগুলি রিপ্লে করে।
- **রিয়েল প্রাইস**: আসল ইটিএফ OHLCV ডেটা (SXRV.DE, CRUDP.PA) ব্যবহার করে — কোনও মার্কিন প্রক্সি নেই।
- **T212 ফি**: Trading 212-এর প্রতি ট্রেডে ০.১% ফি মডেল অনুকরণ করে।
- **বেসলাইন কম্পারিজন**: বেঞ্চমার্ক হিসেবে স্বয়ংক্রিয়ভাবে buy-and-hold পারফরম্যান্স গণনা করে।
- **মেট্রিক্স**: প্রতিটি টিকারের জন্য Sharpe Ratio, Maximum Drawdown, Win Rate, Alpha, Total Return।
