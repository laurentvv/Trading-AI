## 🌟 প্রকল্প সম্পর্কে

NASDAQ এবং তেল (WTI) ETF ট্রেডিংয়ের জন্য একটি বিশেষজ্ঞ সিদ্ধান্ত-সহায়ক ব্যবস্থা, যা ১২-মডেলের হাইব্রিড আর্টিফিশিয়াল ইন্টেলিজেন্স ব্যবহার করে।

### 🚀 ডুয়াল-টিকার কৌশল (বিশ্লেষণ vs. ট্রেডিং)

ব্যবস্থাটি একটি **বৈশ্বিক সূচক বিশ্লেষণ করে** (যেমন Nasdaq-100 এর জন্য `^NDX`, WTI এর জন্য `CL=F`) কিন্তু **EUR-এ মূল্যের একটি ETF-এ সম্পাদন করে** (যেমন `SXRV.DE`, `CRUDP.PA`)। এই পৃথকীকরণ উচ্চ-বিশ্বস্ততার ডেটার উপর বিশ্লেষণ এবং Trading 212-এর মাধ্যমে অ্যাক্সেসযোগ্য সম্পদের উপর প্রকৃত সম্পাদন নিশ্চিত করে।

### 🧠 হাইব্রিড এআই ইঞ্জিন

ইঞ্জিন বিজাতীয় মডেলগুলিকে একটি **ওজনযুক্ত ঐকমত্যে** একত্রিত করে:

1. **Scikit-Learn মডেল** (RandomForest, GradientBoosting, LogisticRegression) — ডেটা লিক এড়াতে `TimeSeriesSplit` দ্বারা যাচাইকৃত। আগ্রাসী পরিমাণগত সিগন্যাল (জ্ঞানীয় ওজনের ২৫%)।
2. **TimesFM 2.5** (Google Research) — সময়-সিরিজ পূর্বাভাসের জন্য ফাউন্ডেশন মডেল।
3. **TensorTrade / PPO** (stable-baselines3) — একটি কাস্টম Gymnasium পরিবেশে Reinforcement Learning এজেন্ট।
4. **Gemma 4 12B** (Ollama) — **পাঠ্য** (ম্যাক্রো/সংবাদ) এবং **দৃশ্য** (প্রযুক্তিগত চার্ট) বিশ্লেষণ; **দ্বি-স্তর JSON প্রতিরক্ষা** সক্রিয় `<|think|>` চিন্তা মোড সত্ত্বেও পরিষ্কার JSON গ্যারান্টি দেয়।
5. **হাইব্রিড সেন্টিমেন্ট বিশ্লেষণ** (Alpha Vantage + AlphaEar + Hyperliquid)।
6. **Vincent Ganne Model** — ভূ-রাজনৈতিক লক (WTI, Brent, গ্যাস, ইউরিয়া, DXY) যা Nasdaq-এর নিচ যাচাই করতে শুধুমাত্র BUY সিগন্যাল তৈরি করে।
7. **OilBenchModel** — WTI-এর জন্য বিশেষায়িত কগনিটিভ মডেল (প্রযুক্তিগত ইন্ডিকেটর + EIA মৌলিক + সেন্টিমেন্ট)।

### 🧘 সিদ্ধান্ত দর্শন: "জ্ঞানীয় সতর্কতা"

কগনিটিভ মডেলগুলি (Gemma 4, সেন্টিমেন্ট, Vincent Ganne) সিদ্ধান্তের ওজনের **৭৫%** ধারণ করে, যেখানে আগ্রাসী পরিমাণগত মডেল শুধুমাত্র **২৫%**। এই ইচ্ছাকৃত অতিরিক্ত ওজন নিশ্চিত করে যে গুণগত প্রসঙ্গ পরিমাণগত সিগন্যালকে পরিমিত করে। একটি সিগন্যাল শুধুমাত্র তখনই সম্পাদিত হয় যখন বৈশ্বিক আস্থা **৪০%** অতিক্রম করে; ২০%–৪০% এর মধ্যে এটি HOLD-এ নামিয়ে আনা হয়।

### ✨ মূল বৈশিষ্ট্য

- **হাইব্রিড ক্লাউড/লোকাল LLM আর্কিটেকচার**: পাঠ্য বিশ্লেষণের জন্য অত্যন্ত বুদ্ধিমান "Frontier Models" (DeepSeek, Claude, Gemini) ব্যবহারে `free-llm-api-keys` একীকরণ, যাতে লোকাল Ollama-তে ১০০% শক্তিশালী ফলব্যাক রয়েছে (যা ভিজ্যুয়াল চার্টের জন্য একচেটিয়া ইঞ্জিন থেকে যায়)।
- **ডুয়াল-টিকার দৃষ্টিভঙ্গি**: সূচক বিশ্লেষণ করুন, ETF ট্রেড করুন।
- **T212 লাইভ মূল্য**: Trading 212 API-এর মাধ্যমে রিয়েল-টাইম EUR মূল্য পুনরুদ্ধার (০.২ সেকেন্ড), yfinance ফলব্যাক এবং parquet ক্যাশের সাথে।
- **Dated Brent স্প্রেড**: Brent Spot (Dated) এবং ফিউচার Brent-এর মধ্যে স্প্রেডের মাধ্যমে ভৌত বাজারের চাপ নিরীক্ষণ।
- **নেটওয়ার্ক স্থিতিস্থাপকতা**: yfinance সার্কিট ব্রেকার আলাদা ট্র্যাকার (info vs. download) সহ, সব নেটওয়ার্ক কলে ১০ সেকেন্ড টাইমআউট।
- **ক্যাশে স্বয়ংক্রিয়-অবৈধকরণ**: Parquet ক্যাশে তার পুরনো হওয়া (> ২ দিন) সনাক্ত করে এবং রিফ্রেশ করতে বাধ্য করে। ম্যানুয়াল ক্লিয়ারের জন্য `refresh_cache.py` ব্যবহার করুন।
- **LLM কল সমান্তরালীকরণ**: স্বাধীন মডেল কল (`text_llm`, `visual_llm`, `search_query`, `timesfm`, `tensortrade`, `grebenkov`) একটি `ThreadPoolExecutor`-এ চলে যাতে Ollama অনুমান I/O-এর সাথে ওভারল্যাপ করে। CPU-তে সমালোচনামূলক পথ সাধারণত ৪–৬ মিনিট যেখানে ক্রমিকভাবে ১০+ মিনিট।
- **২৪ ঘণ্টা সার্চ-কোয়েরি ক্যাশে**: LLM-উৎপাদিত ওয়েব সার্চ কোয়েরি `data_cache/search_queries/<ticker>_<date>_<price-sig>.json`-এ ক্যাশে হয়। কী তারিখ + মূল্য-অ্যাকশন স্বাক্ষর (ক্লোজের লগ২ বাকেটিং + RSI বাকেট) দ্বারা, তাই রেজিম পরিবর্তন এটিকে অবৈধ করে। ফলব্যাক কোয়েরি **কখনো** ক্যাশে হয় না (একটি ক্ষণস্থায়ী Ollama ব্যর্থতা ২৪ ঘণ্টার জন্য ক্যাশে দূষিত করতে পারে না)।
- **কঠোর চক্র টাইমআউট**: প্রতিটি টিকার চক্র ৪০-মিনিট বাজেটে মোড়ানো (`main.py`-এ `CYCLE_TIMEOUT_SECONDS`)। টাইমআউট হলে, কর্মী থ্রেড `shutdown(wait=F)` হয় যাতে পরবর্তী টিকার অবিলম্বে শুরু হয়; মেয়াদোত্তীর্ণ টিকারে HOLD প্রয়োগ হয়। পৃথক futures-এর নিজস্ব প্রতি-টাস্ক টাইমআউট আছে (সার্চ ২৪০ সে, ভিজ্যুয়াল ৩০০ সে, পাঠ্য ২৪০ সে, CPU মডেল প্রতিটি ১৮০ সে, সংবাদ ৯০ সে, ওয়েব ক্রল ৩০ সে)।
- **অরফান-থ্রেড নিরাপত্তা**: চক্র টাইমআউটে, একটি প্রতি-টিকার `threading.Event` সেট হয় যাতে অরফান কর্মী যেকোনো `execute_t212_trade` কলের আগেই বেরিয়ে যায় — ব্যবহারকারীকে "HOLD প্রয়োগ করা হয়েছে" প্যানেল দেখানোর পর বাস্তব-অর্থের ট্রেড প্রতিরোধ করে। একটি প্রতি-টিকার `threading.Lock` আরও T212 অর্ডার স্থাপন সিরিয়ালাইজ করে, শিডিউলার ওভারল্যাপ বা সদৃশ `--ticker` আহ্বানের অধীনে ডাবল-ট্রেড ঝুঁকি দূর করে।
- **LLM ব্যর্থতা সেন্টিনেল**: যখন `_query_ollama` তার সব পুনঃপ্রচেষ্টা শেষ করে, তখন ফলব্যাক ডিক্ট একটি `"failed": True` ফ্ল্যাগ বহন করে যাতে ডাউনস্ট্রিম ঐকমত্য লজিক "মডেল HOLD বেছে নিয়েছে" এবং "মডেল ক্র্যাশ করেছে" পার্থক্য করতে পারে (বর্তমানে প্রচারিত কিন্তু ফিল্টার করা হয়নি — একটি পরিচিত ফলো-আপ)।
- **উন্নত কগনিশন**: **Gemma 4 12B** ব্যবহার, **দ্বি-স্তর JSON প্রতিরক্ষা** সহ:
  1. **সার্ভার-সাইড স্কিমা প্রয়োগ** (`format: SCHEMA_*` যাতে `additionalProperties: false`) — লোড-বহনকারী স্তর; প্রতিটি কল সাইটে Ollama-এর `format` প্যারামিটারের মাধ্যমে পাস করা। স্কিমা `src/llm_client.py`-এ সংজ্ঞায়িত (`SCHEMA_TRADING_DECISION`, `SCHEMA_SEARCH_QUERY`, `SCHEMA_OIL_ALLOCATION`)।
  2. **প্রতিরক্ষামূলক সিস্টেম-প্রম্পট প্রত্যয়** (`"...never add a 'thought' key."`) — অপ্রয়োজনীয়-কিন্তু-ক্ষতিকারক-নয় দ্বিতীয় লাইন, স্কিমা স্তরের যেকোনো ভবিষ্যত প্রতিগমনের বিরুদ্ধে belt-and-braces হিসাবে রাখা হয়েছে।

  `<|think|>` চিন্তা টোকেন চারটি প্রোডাকশন সিস্টেম প্রম্পটে **সক্রিয়** (২০২৬-০৬-০৬ তারিখে `think-mode` শাখায় যাচাইয়ের পর `main`-এ পুনরায় সক্রিয়)। স্কিমা স্তরই আসলে ঐতিহাসিক `<|channel>thought` JSON-ধ্বংসসামগ্রী ত্রুটি নিরসন করে (মে ২০২৬ মূল কারণ): `tests/check_llm_json.py` নিশ্চিত করে যে schema-strict ক্ষেত্রে (`v3_schema`, `v6_schema`, `v7_schema_strict`) `<|think|>` সক্রিয় থাকা সত্ত্বেও পরিষ্কার JSON উৎপন্ন হয়, যেখানে loose `format:json` ভেরিয়েন্ট ব্যর্থ হয়। সম্পূর্ণ বিশ্লেষণ ও বিপরীত প্রক্রিয়ার জন্য `docs/ADR-001-think-mode-dual-layer-defence.md` দেখুন।
- **স্বায়ত্তশাসিত Morning Brief এজেন্ট**: একটি রাতের `smolagents`-ভিত্তিক ওয়ার্কফ্লো (`morning_brief/morning_brief.py`) যা `schedule.py`-এর মাধ্যমে স্বয়ংক্রিয়ভাবে প্রতিদিন ০১:০০-এ চলে। এটি স্বাধীনভাবে দৈনিক API লগ ক্রল করে, মৌলিক EIA ইনভেন্টরি ডেটা ডাউনলোড করে, এবং একটি *Bull vs Bear* বিতর্কের মধ্যস্থতা করে। ফলস্বরূপ markdown রিপোর্ট (`morning_market_brief.md`) দৈনিক ট্রেডিং চক্রের সময় পাঠ্য LLM-এর সিস্টেম প্রম্পটে স্বয়ংক্রিয়ভাবে ইনজেক্ট হয়, লাইভ বাজার সম্পাদন ধীর না করেই প্রধান এআই-কে গভীর প্রাসঙ্গিক স্মৃতি ও মৌলিক সচেতনতা প্রদান করে।

### 🧠 এআই ও LLM আর্কিটেকচার (Gemini + লোকাল ফলব্যাক)
ব্যবস্থাটি সর্বোচ্চ আপটাইম ও বুদ্ধিদীপ্ত সিদ্ধান্ত গ্রহণ নিশ্চিত করতে একটি অত্যন্ত শক্তিশালী, বহু-স্তর আর্কিটেকচার ব্যবহার করে, যা `main.py` ও Weekend Council-এ গভীরভাবে একীকৃত।

- **৪-স্তর ক্যাসকেড ফলব্যাক**:
  1. **Gemini পেইড স্তর (`GEMINI_API_KEY_PAY`)**: সর্বোচ্চ অগ্রাধিকার। জটিল যুক্তি, প্রযুক্তিগত চার্ট ভিশন ও চূড়ান্ত ট্রেড সিদ্ধান্তের জন্য Gemini 2.5 Pro-এর মতো উন্নত মডেল ব্যবহার করে।
  2. **Gemini ফ্রি স্তর (`GEMINI_API_KEY`)**: ওয়েব প্রসঙ্গ সারসংক্ষেপের মতো হালকা, উচ্চ-আয়তনের কাজের জন্য ব্যবহৃত।
  3. **ফ্রি LLM API প্রক্সি**: `free-llm-api-keys`-এর মাধ্যমে ব্যাকআপ।
  4. **লোকাল Ollama**: সব ক্লাউড পরিষেবা ব্যর্থ হলে ১০০% শক্তিশালী, অফলাইন CPU ফলব্যাক।
- **খরচ সুরক্ষা**: পেইড স্তর একটি রোলিং ৩০-দিনের খরচ বাজেট (`GEMINI_PAY_MONTHLY_BUDGET_EUR`, ডিফল্ট 8.6 €/মাস) দ্বারা সীমাবদ্ধ — প্রতিটি কলের খরচ প্রকৃত token ব্যবহার × মডেলের দাম থেকে গণনা করে জমা করা হয়; বাজেটে পৌঁছালে কলগুলি ফ্রি স্তর / Ollama-তে ফিরে যায়। একটি দৈনিক backstop (`GEMINI_PAY_DAILY_CAP`, ডিফল্ট 200) অনিয়ন্ত্রিত লুপ থেকে রক্ষা করে।
- **একীকরণ**: প্রধান দৈনিক সম্পাদন ইঞ্জিন (`main.py`) রিয়েল-টাইম বহু-মডেল ঐকমত্যের জন্য Gemini ব্যবহার করে, যখন অসিঙ্ক্রোনাস Weekend Council (`council`) বিচিত্র লোকাল Ollama মডেলের পাশাপাশি নির্দিষ্ট ভূমিকার (যেমন Judge ও Sceptique) জন্য Gemini একীকরণ করে।

## 🚀 দ্রুত শুরু

আপনার লোকাল ডেভেলপমেন্ট পরিবেশ সেটআপ করতে এই ধাপগুলি অনুসরণ করুন।

### ✅ পূর্বশর্ত

- Python 3.12+ (`uv`-এর মাধ্যমে)
- [Ollama](https://ollama.com/) ইনস্টল করা এবং লোকালি চলমান।
- ডাউনলোড করা LLM মডেল: `ollama pull hf.co/unsloth/gemma-4-12b-it-GGUF:Q6_K`
- **Weekend Council মডেল** (ঐচ্ছিক, কিন্তু council-এর যুক্তি বৈচিত্র্যের জন্য প্রয়োজন): council প্রতিটি পার্সোনাকে একটি *ভিন্ন* মডেল পরিবারে (Gemma / GLM / Qwen / LFM) চালায়। এগুলি সব একসাথে `uv run python setup_council_models.py` দিয়ে ইনস্টল করুন।

### ⚙️ ইনস্টলেশন

1.  **রিপোজিটরি ক্লোন করুন:**
    ```sh
    git clone https://github.com/laurentvv/Trading-AI.git
    cd Trading-AI
    ```
2.  **`uv` ইনস্টল করুন (যদি ইতিমধ্যে না করে থাকেন):**
    ইনস্টলেশন নির্দেশাবলীর জন্য [astral.sh/uv](https://astral.sh/uv) দেখুন।

3.  **ভার্চুয়াল পরিবেশ তৈরি ও সক্রিয় করুন (অত্যন্ত গুরুত্বপূর্ণ ধাপ):**
    আপনাকে ফাউন্ডেশন মডেল ইনস্টল করার আগে `.venv` তৈরি করতে ও সক্রিয় করতে হবে।
    ```bash
    uv venv
    source .venv/bin/activate  # Windows-এ, `.\.venv\Scripts\activate.ps1` ব্যবহার করুন
    ```

4.  **ফাউন্ডেশন মডেল ইনস্টল করুন:**
    মডেলগুলিকে `vendor/`-এ ক্লোন করতে ও প্যাচ প্রয়োগ করতে ইনস্টলেশন স্ক্রিপ্ট চালান:
    ```bash
    python setup_timesfm.py
    ```

5.  **পরিবেশ শুরু ও সিঙ্ক করুন:**
    ```bash
    uv sync
    ```

6.  **ওয়েব গবেষণার জন্য ব্রাউজার ইনস্টল করুন (Crawl4AI):**
    ```bash
    uv run python -m playwright install chromium
    ```

7.  **আপনার API কী কনফিগার করুন:**
    প্রকল্প রুটে একটি `.env` ফাইল তৈরি করুন:
    ```
    ALPHA_VANTAGE_API_KEY="আপনার_কী"
    EIA_API_KEY="আপনার_কী"

    # ঐচ্ছিক কিন্তু অত্যন্ত সুপারিশকৃত: Gemini AI একীকরণ
    GEMINI_API_KEY_PAY="আপনার_পেইড_স্তর_কী"  # জটিল যুক্তি/ভিশনের জন্য (Gemini 2.5 Pro)
    GEMINI_API_KEY="আপনার_ফ্রি_স্তর_কী"      # হালকা কাজের জন্য (সারসংক্ষেপ)
    GEMINI_PAY_MONTHLY_BUDGET_EUR=8.6        # রোলিং ৩০-দিনের খরচ বাজেট (€) — লোড-বহনকারী বিলিং গার্ড
    GEMINI_PAY_DAILY_CAP=200                 # ব্যাকস্টপ: প্রতি দিন সর্বোচ্চ পেইড API কল
    ```

## 🛠️ ব্যবহার

ব্যবস্থাটি প্রতিটি সম্পাদনে সর্বশেষ ডেটার উপর তার মডেলগুলি প্রশিক্ষণ দেয়, তারপর একটি সিদ্ধান্ত দেয়।

### সিমুলেশন মোড (Paper Trading)

১০০০ €-এর কাল্পনিক মূলধন নিয়ে ঝুঁকি ছাড়া ব্যবস্থাটি পরীক্ষা করতে `--simul` ফ্ল্যাগ ব্যবহার করুন। ব্যবস্থাটি ক্রয় ও বিক্রয়ের একটি কঠোর ইতিহাস পরিচালনা করবে।

```sh
# একটি সিমুলেটেড বিশ্লেষণ চালান (ডিফল্ট: SXRV.DE - Nasdaq 100 EUR)
uv run main.py --simul

# তেলে (WTI) চালান
uv run main.py --ticker CRUDP.PA --simul
```

### প্রকৃত সম্পাদন (Trading 212)

ব্যবস্থাটি এখন Trading 212-এর সাথে **সম্পূর্ণভাবে একীকৃত**:
- **পোর্টফোলিও যাচাই**: যেকোনো পদক্ষেপের আগে, রোবট আপনার প্রকৃত নগদ ও অবস্থান পরীক্ষা করে।
- **API ব্যবস্থাপনা**: অনুরোধ সীমার (Rate Limiting) বিরুদ্ধে স্বয়ংক্রিয় পুনঃপ্রচেষ্টা প্রক্রিয়া অন্তর্ভুক্ত।

```sh
# প্রকৃত সম্পাদন সহ বিশ্লেষণ চালান (Demo বা Real .env অনুযায়ী)
uv run main.py --t212
```

## 🧪 প্রোডাকশন ব্যাকটেস্টিং

ব্যবস্থায় একটি **স্ট্যান্ডঅ্যালোন প্রোডাকশন ব্যাকটেস্ট ইঞ্জিন** (`backtest_prod.py`) অন্তর্ভুক্ত যা `logs_prod/trading_journal.csv` থেকে প্রকৃত prod সিগন্যালকে `data_cache/` Parquet ফাইলের প্রকৃত মূল্যের বিরুদ্ধে রিপ্লে করে।

### বৈশিষ্ট্য
- **প্রকৃত সিগন্যাল**: ১২-মডেল হাইব্রিড ইঞ্জিনের সঠিক সিদ্ধান্ত রিপ্লে করে।
- **প্রকৃত মূল্য**: ETF-গুলির প্রকৃত OHLCV ডেটা (SXRV.DE, CRUDP.PA) ব্যবহার করে — কোনো US প্রক্সি নেই।
- **T212 ফি**: প্রতি-ট্রেডে ০.১% Trading 212 ফি মডেল সিমুলেট করে।
- **বেসলাইন তুলনা**: বেঞ্চমার্ক হিসাবে buy-and-hold কর্মক্ষমতা স্বয়ংক্রিয়ভাবে গণনা করে।
- **মেট্রিক্স**: প্রতি টিকারে Sharpe Ratio, সর্বোচ্চ Drawdown, Win Rate, Alpha, মোট প্রতিফল।

### ব্যবহার

```bash
uv run python backtest_prod.py
```

ফলাফল `logs_prod/backtest_report.json`-এ CSV ইক্যুইটি বক্ররেখা সহ সংরক্ষিত হয়।
