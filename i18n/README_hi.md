## 🌟 परियोजना के बारे में

यह परियोजना ETF ट्रेडिंग के लिए एक विशेषज्ञ निर्णय-समर्थन प्रणाली है, जो त्रि-मोडल हाइब्रिड AI दृष्टिकोण का उपयोग करती है। इसे कई AI दृष्टिकोणों को मिलाकर एक व्यापक और मजबूत विश्लेषण प्रदान करने के लिए डिज़ाइन किया गया है।

### 🚀 डुअल-टिकर रणनीति (विश्लेषण बनाम ट्रेडिंग)
प्रणाली मॉडल की सटीकता को अधिकतम करने के लिए एक अभिनव दृष्टिकोण का उपयोग करती है:
- **हाई-फिडेलिटी विश्लेषण**: AI मॉडल **वैश्विक संदर्भ सूचकांकों** (Nasdaq के लिए `^NDX`, WTI कच्चे तेल के लिए `CL=F`) का विश्लेषण करते हैं। ये सूचकांक लंबी ऐतिहासिक जानकारी और "शुद्ध" रुझान प्रदान करते हैं, जिसमें ट्रेडिंग घंटों या ETF शुल्क से संबंधित शोर नहीं होता है।
- **ETF निष्पादन**: वास्तविक ऑर्डर **Trading 212** पर संबंधित टिकर (`SXRV.DE`, `CRUDP.PA`) पर दिए जाते हैं, जिसमें पोजीशन साइजिंग के लिए **T212 लाइव कीमतों** (पोजीशन API के माध्यम से) का उपयोग किया जाता है।

### 🧠 हाइब्रिड AI इंजन
प्रणाली आठ अलग-अलग संकेतों को मिलाती है:
1.  **क्लासिक क्वांटिटेटिव मॉडल**: तकनीकी और व्यापक आर्थिक संकेतकों पर प्रशिक्षित RandomForest/GradientBoosting/LogisticRegression पहनावा।
2.  **TimesFM 2.5 (Google Research)**: समय-श्रृंखला पूर्वानुमान के लिए अत्याधुनिक फाउंडेशन मॉडल।
3.  **ऑयल-बेंच मॉडल (Gemma 4:e4b)**: ऊर्जा-विशेषज्ञ मॉडल जो **EIA** मौलिक डेटा (स्टॉक, आयात, रिफाइनरी उपयोग) और WTI ट्रेडिंग के लिए भावना को जोड़ता है।
4.  **टेक्स्टुअल LLM (Gemma 4:e4b)**: कच्चे डेटा का प्रासंगिक विश्लेषण, **AlphaEar** कौशल के माध्यम से वास्तविक समय की खबरें, और गतिशील **व्यापक-आर्थिक वेब अनुसंधान** का एकीकरण।
5.  **विजुअल LLM (Gemma 4:e4b)**: तकनीकी चार्ट (`enhanced_trading_chart.png`) का प्रत्यक्ष विश्लेषण।
6.  **भावना विश्लेषण**: अल्फा वांटेज और **AlphaEar** (Weibo, WallstreetCN) के "हॉट" रुझानों के संयोजन वाला हाइब्रिड विश्लेषण।
7.  **विकेंद्रीकृत डेटा (Hyperliquid)**: *फंडिंग रेट* और *ओपन इंटरेस्ट* के माध्यम से तेल (WTI) पर सट्टा भावना का विश्लेषण।
8.  **विंसेंट गने मॉडल**: व्यापक आर्थिक गिरावट का पता लगाने के लिए भू-राजनीतिक और क्रॉस-एसेट विश्लेषण (WTI, ब्रेंट, गैस, DXY, MA200)।

लक्ष्य **सटीकता प्रथम** की पूर्ण प्राथमिकता के साथ अंतिम निर्णय (`BUY`, `SELL`, `HOLD`) देना है।

### 🧘 निर्णय दर्शन: "संज्ञानात्मक सावधानी"
क्लासिक ट्रेडिंग एल्गोरिदम के विपरीत, जो अस्थिरता के विस्फोट होते ही घबरा जाते हैं, यह प्रणाली एक सूचित निवेशक दृष्टिकोण लागू करती है:
- **मजबूत आम सहमति आवश्यक**: एक मात्रात्मक मॉडल (क्लासिक) चेतावनी (`SELL`) दे सकता है, लेकिन यदि संज्ञानात्मक मॉडल (टेक्स्ट LLM, विजन, TimesFM) तटस्थ रहते हैं, तो प्रणाली `HOLD` को प्राथमिकता देगी।
- **आत्मविश्वास फ़िल्टर**: एक गति निर्णय (खरीदें या बेचें) तभी मान्य होता है जब वैश्विक आत्मविश्वास सुरक्षा सीमा (आमतौर पर 40%) से अधिक हो जाता है। इससे नीचे, प्रणाली संकेत को "शोर" मानती है और स्टैंडबाय पर रहती है।
- **पूंजी सुरक्षा**: `VERY_HIGH` जोखिम मोड में, `HOLD` एक ढाल के रूप में कार्य करता है। यह एक अस्थिर बाजार में प्रवेश करने से रोकता है और एक साधारण तकनीकी सुधार पर समय से पहले बाहर निकलने से रोकता है यदि मौलिक (समाचार/विजन/हाइपरलिक्विड) आसन्न गिरावट की पुष्टि नहीं करते हैं।

### ✨ मुख्य विशेषताएं

- **डुअल-टिकर दृष्टिकोण**: सूचकांक का विश्लेषण करें, ETF का व्यापार करें।
- **T212 लाइव कीमतें**: Trading 212 API (0.2s) के माध्यम से वास्तविक समय में EUR कीमतों की रिकवरी, yfinance फ़ॉलबैक और पैरक्वेट कैश के साथ।
- **Dated Brent स्प्रेड**: ब्रेंट स्पॉट (Dated) और ब्रेंट फ्यूचर्स के बीच स्प्रेड के माध्यम से भौतिक बाजार के तनाव की निगरानी।
- **नेटवर्क लचीलापन**: अलग-अलग ट्रैकर्स (जानकारी बनाम डाउनलोड) के साथ yfinance सर्किट ब्रेकर, सभी नेटवर्क कॉल पर 10s टाइमआउट।
- **उन्नत संज्ञान**: बेहतर तकनीकी/मौलिक संश्लेषण के लिए **Gemma 4** का उपयोग।
- **समाचार और ब्लॉकचेन भावना**: सामाजिक और सट्टा भावना को पकड़ने के लिए **AlphaEar** और **Hyperliquid** का एकीकरण।
- **स्वचालित शेड्यूलर**: सर्वर पर निरंतर निष्पादन (8:30 AM - 6:00 PM) के लिए `schedule.py` स्क्रिप्ट।
- **उन्नत जोखिम प्रबंधन**: अस्थिरता और बाजार व्यवस्था के आधार पर स्वचालित संकेत समायोजन।

### 💻 टेक स्टैक

- **भाषा**: `Python 3.12+`
- **गणना और डेटा**: `pandas`, `numpy`, `yfinance`, `pyarrow`, `pandas_datareader`, `hyperliquid-python-sdk`
- **मशीन लर्निंग**: `scikit-learn`, `shap`
- **AI और LLM**: `requests`, `ollama`
- **वेब स्क्रैपिंग और खोज**: `beautifulsoup4`, `duckduckgo_search`, `crawl4ai`
- **विज़ुअलाइज़ेशन**: `matplotlib`, `seaborn`, `mplfinance`
- **उपयोगिताएँ**: `tqdm`, `rich`, `python-dotenv`, `schedule`

### ⚙️ प्रदर्शन और हार्डवेयर
प्रणाली को समर्पित GPU की आवश्यकता के बिना **उपभोक्ता हार्डवेयर पर प्रदर्शन करने के लिए** डिज़ाइन किया गया है।
- **केवल CPU**: LLM अनुमान (ओलामा के माध्यम से Gemma 4) और TimesFM तेजी से CPU निष्पादन के लिए अनुकूलित हैं यदि पर्याप्त RAM उपलब्ध है।
- **अनुशंसित RAM**: न्यूनतम 16 GB (Gemma 4 को आराम से चलाने के लिए 32 GB का सुझाव दिया गया है)।
- **निष्पादन समय**: एक पूर्ण चक्र के लिए ~2 से 5 मिनट (वेब क्रॉलिंग, ML प्रशिक्षण, TimesFM भविष्यवाणियों और 3 LLM विश्लेषणों सहित)।
- **API गति**: अल्ट्रा-फास्ट ट्रेडिंग 212 एकीकरण (लाइव मूल्य रिकवरी के लिए <1s)।

## 🚀 त्वरित शुरुआत

अपने स्थानीय विकास वातावरण को स्थापित करने के लिए इन चरणों का पालन करें।

### ✅ पूर्व-आवश्यकताएँ

- Python 3.12+ (`uv` के माध्यम से)
- [Ollama](https://ollama.com/) स्थानीय रूप से स्थापित और चल रहा है।
- डाउनलोड किया गया LLM मॉडल: `ollama pull gemma4:e4b`

### ⚙️ स्थापना

1.  **रिपॉजिटरी क्लोन करें:**
    ```sh
    git clone https://github.com/laurentvv/Trading-AI.git
    cd Trading-AI
    ```
2.  **`uv` स्थापित करें (यदि पहले से नहीं किया है):**
    स्थापना निर्देशों के लिए [astral.sh/uv](https://astral.sh/uv) देखें।

3.  **TimesFM 2.5 को स्थापित और पैच करें (महत्वपूर्ण चरण):**
    मॉडल को `vendor/` में क्लोन करने और पैच लागू करने के लिए स्थापना स्क्रिप्ट चलाएँ:
    ```bash
    python setup_timesfm.py
    ```

4.  **वातावरण को प्रारंभ और सिंक्रनाइज़ करें:**
    ```bash
    uv sync
    ```

5.  **वेब अनुसंधान के लिए ब्राउज़र स्थापित करें (Crawl4AI):**
    ```bash
    uv run python -m playwright install chromium
    ```

6.  **अपनी API कुंजियाँ कॉन्फ़िगर करें:**
    परियोजना रूट में एक `.env` फ़ाइल बनाएँ:
    ```
    ALPHA_VANTAGE_API_KEY="YOUR_KEY"
    EIA_API_KEY="YOUR_KEY"
    ```

## 🛠️ उपयोग

प्रणाली निर्णय देने से पहले प्रत्येक निष्पादन पर नवीनतम डेटा पर अपने मॉडल को प्रशिक्षित करती है।

### सिमुलेशन मोड (पेपर ट्रेडिंग)

बिना जोखिम के €1000 की काल्पनिक पूंजी के साथ प्रणाली का परीक्षण करने के लिए, `--simul` ध्वज का उपयोग करें। प्रणाली खरीद और बिक्री के सख्त इतिहास का प्रबंधन करेगी।

```sh
# सिमुलेशन विश्लेषण चलाएँ (डिफ़ॉल्ट: SXRV.DE - Nasdaq 100 EUR)
uv run main.py --simul

# तेल (WTI) पर चलाएँ
uv run main.py --ticker CRUDP.PA --simul
```

### वास्तविक निष्पादन (Trading 212)

प्रणाली अब Trading 212 के साथ **पूरी तरह से एकीकृत** है:
- **पोर्टफोलियो सत्यापन**: किसी भी कार्रवाई से पहले, रोबोट आपकी वास्तविक नकदी और पोजीशन से परामर्श करता है।
- **API प्रबंधन**: अनुरोध सीमाओं (दर सीमित करना) के खिलाफ स्वचालित पुन: प्रयास तंत्र शामिल है।

```sh
# वास्तविक निष्पादन के साथ विश्लेषण चलाएँ (.env के अनुसार डेमो या रियल)
uv run main.py --t212
```
