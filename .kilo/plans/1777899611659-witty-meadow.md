### 1. 🔴 BUG — `run_lean_backtest.py` : `--export-signals` seul déclenche un backtest
**Fichier** : `run_lean_backtest.py:43-81`

**Problème** : La structure `if / elif / else` fait que `--export-signals` seul passe dans le `else` (ligne 73) et lance un backtest par défaut via `validator.run_backtest()`.

**Correction** : Restructurer en `if/elif/elif` avec un early return après l'export, ou ajouter une condition `elif not args.export_signals:` avant le `else`.

```python
# Avant (bugué) :
if args.export_signals:
    ...
validator = LeanValidator()
if args.compare:
    ...
elif args.validate:
    ...
else:                          # ← exécuté même si --export-signals seul
    validator.run_backtest(...)

# Après (corrigé) :
if args.export_signals:
    ...
    if not args.validate and not args.compare:
        return                 # ← early exit, pas de backtest

validator = LeanValidator()
if args.compare:
    ...
elif args.validate or args.export_signals:
    ...
else:
    ...
```

### 2. 🟡 BUG — `_parse_confidence` utilisé pour un montant EUR (`T212_Capital`)
**Fichier** : `src/lean_bridge.py:129`

**Problème** : `T212_Capital` est un montant en euros (ex: `1000.50€`). La fonction `_parse_confidence` divise par 100 si la valeur > 1.0, transformant `1000€` → `10.0`.

**Correction** : Créer une fonction `_parse_currency` séparée qui supprime `%` et `€` mais ne divise pas par 100.

```python
def _parse_currency(value) -> float:
    if isinstance(value, (int, float)):
        return float(value)
    if isinstance(value, str):
        cleaned = value.replace("€", "").replace(",", ".").strip()
        try:
            return float(cleaned)
        except ValueError:
            return 0.0
    return 0.0
```

Et remplacer la ligne 129 :
```python
"t212_capital": _parse_currency(row.get("T212_Capital", 0)),
```

### 4. 🟡 CONCEPT — `SentimentAlphaModel` toujours FLAT (modèle mort)
**Fichier** : `TradingAI-Lean/AlphaModels/TradingAICompositeAlpha.py:111-136`

**Problème** : `SentimentAlphaModel` n'a aucun mécanisme d'update dans le framework. `_last_signal` reste toujours `FLAT`. Il émet des insights à 0.3 de confidence qui diluent le composite.

**Correction** : Implémenter un signal basé sur la volatilité récente (proxy simple du sentiment — hausse de volatilité = sentiment négatif).

```python
class SentimentAlphaModel(AlphaModel):
    def __init__(self, lookback: int = 20, vol_threshold: float = 1.5):
        self.lookback = lookback
        self.vol_threshold = vol_threshold
        self.name = "SentimentAlpha"

    def update(self, algorithm, data):
        insights = []
        for symbol in algorithm.securities.keys():
            history = algorithm.history(symbol, self.lookback, Resolution.DAILY)
            if history.empty or len(history) < 10:
                continue
            close = history["close"]
            returns = close.pct_change().dropna()
            recent_vol = returns.tail(5).std()
            baseline_vol = returns.std()
            vol_ratio = recent_vol / baseline_vol if baseline_vol > 0 else 1.0

            if vol_ratio > self.vol_threshold:
                direction = InsightDirection.DOWN
                confidence = min(0.7, vol_ratio * 0.3)
            elif vol_ratio < 0.5:
                direction = InsightDirection.UP
                confidence = min(0.6, (1.0 - vol_ratio) * 0.5)
            else:
                direction = InsightDirection.FLAT
                confidence = 0.2

            insights.append(Insight.price(symbol, timedelta(days=5), direction, 0.02, confidence, source_model=self.name))
        return insights
```

**Correction** (optionnelle, priorité basse) : Pré-calculer les DataFrames d'historique une fois dans le composite et les passer aux sous-modèles.
