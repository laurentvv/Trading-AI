# Plan : Utiliser Trading 212 comme source de prix ETF + optimisation data.py

## Distinction critique : INDEX vs ETF

| Usage | Ticker | Source | T212 ? |
|---|---|---|---|
| Analyse technique IA | `^NDX` (Nasdaq), `CL=F` (Crude) | yfinance + cache parquet | **NON** - T212 n'a pas ces indices |
| Prix actuel ETF | `SXRV.DE`, `CRUDP.PA` | yfinance + cache parquet | **OUI** - T212 a le prix live |
| Passage d'ordres | `SXRVd_EQ`, `CRUDl_EQ` | T212 API | OUI |

## Flux de prix actuels dans le code

1. **`prepare_data_and_features()`** (`enhanced_trading_example.py:114-118`) :
   - `get_etf_data(self.analysis_ticker)` → INDEX → yfinance/cache uniquement
   - `get_etf_data(self.ticker)` puis `.iloc[-1]` → ETF → **T212 peut remplacer pour le prix live**

2. **`execute_t212_trade()`** (`t212_executor.py:340-345`) :
   - `get_real_price_eur(ticker)` → ETF → **T212 peut etre primaire**
   - `get_real_price_eur(index_ticker)` → INDEX → yfinance uniquement

3. **`get_etf_data()`** (`data.py:105+`) :
   - Charge le cache parquet (rapide)
   - Appelle `_yf_ticker_info()` 3x (lent, inutile) → **a skipper**

## Changements

### 1. Nouvelle fonction `get_t212_price()` dans `t212_executor.py`

Lit `GET /equity/positions` pour trouver le prix live de l'ETF.
Si pas de position ouverte, retourne None et on fallback sur yfinance.

**Limitation** : T212 ne donne le prix que si une position existe. Pour SXRV.DE (pas de position actuellement), le fallback yfinance sera utilise. Pour CRUDL_EQ (position ouverte), T212 donnera le prix live.

### 2. Modifier `get_real_price_eur()` : T212 en premier pour les ETF

Nouveau flow :
1. `get_t212_price(target)` → si position T212 ouverte
2. `MarketDataManager.get_price_data()` → yfinance avec timeout 10s
3. `yf.Ticker().history(period="5d")` → dernier recours

### 3. Modifier `prepare_data_and_features()` pour prix ETF live

Actuellement ligne 117-118 :
```python
etf_data, _ = get_etf_data(ticker=self.ticker)
current_etf_price = etf_data['Close'].iloc[-1]
```

Remplacer par :
```python
t212_price = get_t212_price(self.ticker)
if t212_price:
    current_etf_price = t212_price
else:
    etf_data, _ = get_etf_data(ticker=self.ticker)
    current_etf_price = etf_data['Close'].iloc[-1]
```

### 4. Separer les circuit breakers dans `data.py`

- `_yf_info_tracker` : pour `_yf_ticker_info()` (metadata non-critique)
- `_yf_download_tracker` : pour `_yf_download()` (donnees d'analyse, Vincent Ganne, etc.)

Les timeouts de `info` ne bloqueront plus les `download` de VG.

### 5. Skip `_yf_ticker_info()` quand le cache existe

Dans `get_etf_data()`, le `info` (metadata Yahoo : PEG ratio, etc.) n'est jamais utilise dans le pipeline d'analyse. Remplacer `info = _yf_ticker_info(ticker)` par `info = {}` quand on charge depuis le cache. Gain : -30 a -50s par cycle.
