# Plan: Feature Ideas Inspired by daily_stock_analysis

## Context

**daily_stock_analysis (DSA)** — 32k stars, LLM-powered A/H/US stock analysis with Web UI, multi-channel push, market review, agent chat, backtesting, and 7+ news sources.

**Trading-AI** — Hybrid 9-model AI system for ETF trading (NASDAQ-100 + WTI), with real Trading 212 execution, TimesFM, RL, EIA data, but CLI-only, 2 tickers, no Web UI, no notifications.

---

## Feature Ideas (by priority/impact)

### P0 — High Impact, Feasible

#### 1. Multi-Channel Notifications (Telegram / Discord / Email)
**DSA has:** WeChat, Feishu, Telegram, Discord, Slack, Email push after each analysis cycle.
**Trading-AI gap:** `performance_monitor.py` has email scaffolding (`smtplib` imported) but nothing active. Zero push notifications.
**Proposal:** Add a `src/notifier.py` module with:
- Telegram Bot push (via `python-telegram-bot` or raw HTTP)
- Discord Webhook
- Email (finish existing scaffolding)
- Push a formatted summary after each scheduler cycle: decision, confidence, risk level, model signals table
- Configurable via `.env` (`TELEGRAM_BOT_TOKEN`, `DISCORD_WEBHOOK_URL`, etc.)

#### 2. FastAPI Web Dashboard
**DSA has:** Full FastAPI Web UI with dual themes, manual analysis, history, backtesting, portfolio management.
**Trading-AI gap:** Entirely CLI-based (Rich console). Static PNG dashboards only.
**Proposal:** Add a `src/web/` module with FastAPI:
- `/` — Dashboard: current positions, latest signals, P&L chart
- `/history` — Past decisions with filtering
- `/config` — View/edit `.env` settings
- `/api/signals` — JSON endpoint for latest model signals
- WebSocket for real-time cycle progress during scheduler runs
- Use Jinja2 templates + HTMX (lightweight, no npm build step needed)
- Start with `--webui` flag like DSA

#### 3. Decision Dashboard / Summary Report
**DSA has:** "决策仪表盘" — one-line conclusion + score + buy/sell levels + risk alerts + action checklist.
**Trading-AI gap:** The `EnhancedTradingSystem` outputs verbose Rich tables but no concise executive summary.
**Proposal:** Add a `src/report_generator.py` that produces:
- One-line verdict: `BUY SXRV.DE | Confidence 78% | Target €28.50 | Stop €25.80`
- Risk alerts list (from `advanced_risk_manager.py`)
- Catalyst events (from `web_researcher.py` + `news_fetcher.py`)
- Model consensus breakdown (compact table)
- Markdown + plain-text formats (for push notifications)
- Save to `reports/YYYY-MM-DD_HHMM.md`

#### 4. Extended Ticker Support (Watchlist)
**DSA has:** User-defined stock list, supports A/H/US markets simultaneously.
**Trading-AI gap:** Hardcoded 2 tickers (`SXRV.DE` / `CRUDP.PA`).
**Proposal:** Make ticker configuration dynamic:
- `TICKER_LIST=SNRD.DE,CRUDP.PA,CSPX.L,XDWD.DE` in `.env`
- Each ticker gets analyzed independently per cycle
- Cross-ticker correlation analysis (portfolio-level view)
- Reference index auto-mapping (e.g., `SXRV.DE` -> `^NDX`, `CRUDP.PA` -> `CL=F`)
- Keep existing dual-index strategy as default, allow expansion

### P1 — Medium Impact, Moderate Effort

#### 5. Market Review / 大盘复盘
**DSA has:** Daily market overview: index performance, sector strength, advance/decline stats.
**Trading-AI gap:** No broad market context before trading decisions.
**Proposal:** Add `src/market_review.py`:
- Fetch major indices: S&P 500, NASDAQ, DAX, CAC40, WTI, Brent, VIX, DXY
- Compute daily change, weekly trend, RSI
- Sector ETF performance (XLK, XLE, XLF, etc.)
- Fear & Greed indicator approximation
- Push as a pre-cycle briefing before the 9-model analysis

#### 6. Historical Report Management & Replay
**DSA has:** Browse, search, re-analyze historical reports.
**Trading-AI gap:** `trading_journal.csv` exists but no query/replay UI.
**Proposal:**
- Store full analysis context in SQLite (not just CSV)
- API endpoints: `GET /reports?date=...&ticker=...`
- "Re-analyze" button: re-run models on historical data with current config
- Compare past decisions vs actual outcomes (accuracy tracking)

#### 7. AI Backtesting Verification
**DSA has:** Post-hoc verification of historical analyses — check if predictions were correct.
**Trading-AI gap:** Backtests exist but are script-based, no automated accuracy tracking.
**Proposal:**
- After each cycle, log: predicted direction + confidence + actual price at T+1, T+5, T+10
- Weekly/monthly accuracy report per model
- Confusion matrix per model (predicted UP/DOWN vs actual)
- Auto-adjust model weights based on accuracy (enhance `adaptive_weight_manager.py`)

#### 8. Multi-LLM Support (Beyond Ollama)
**DSA has:** Gemini, OpenAI, Claude, DeepSeek, Ollama via LiteLLM routing.
**Trading-AI gap:** Only Ollama/Gemma 4 local. Slow (~60s/call).
**Proposal:**
- Add `litellm` or direct API clients for OpenAI, Claude, Gemini
- Configurable model per task: e.g., Gemini for news analysis (fast), Ollama for chart vision (local)
- Fallback chain: cloud API -> local Ollama -> rule-based
- `LLM_PROVIDER=openai|claude|gemini|ollama` in `.env`

#### 9. GitHub Actions CI/CD
**DSA has:** Full GitHub Actions with scheduled runs, CI gates, Docker.
**Trading-AI gap:** `.pre-commit-config.yaml` is empty. No CI. No automated tests.
**Proposal:**
- Add `.github/workflows/ci.yml`: ruff lint + pytest on push/PR
- Add `.github/workflows/scheduled-analysis.yml`: run `main.py --simul` on schedule
- Dockerfile for reproducible deployment
- Fix pre-commit hooks

### P2 — Nice-to-Have, Larger Scope

#### 10. Agent Chat for Strategy Q&A
**DSA has:** Multi-turn strategy conversations with 11 built-in strategies.
**Proposal:**
- Add `/chat` endpoint in the Web UI
- LLM-powered Q&A about current positions, market conditions
- Tool-use: agent can call `data.py`, `features.py`, `chart_generator.py` on demand
- "Should I hold or sell SXRV.DE today?" -> agent pulls live data + runs analysis

#### 11. Multi-Strategy System
**DSA has:** 均线金叉, 缠论, 波浪理论, 情绪周期 etc.
**Trading-AI gap:** Strategies are embedded in models (classic_model, oil_bench, etc.), not pluggable.
**Proposal:**
- Create `src/strategies/` with pluggable strategy interface
- Each strategy returns a signal + confidence
- Strategies: MA crossover, RSI extremes, Bollinger breakout, trend following
- Strategy signals feed into the 9-model fusion engine as additional inputs

#### 12. Portfolio-Level Optimization
**DSA has:** Portfolio management in Web UI.
**Trading-AI gap:** Each ticker analyzed independently, no cross-asset allocation.
**Proposal:**
- Correlation matrix between holdings
- Kelly criterion at portfolio level (already per-position in `advanced_risk_manager.py`)
- Risk parity / equal weight suggestions
- Max drawdown tracking across portfolio

#### 13. Order Types Beyond Market
**DSA has:** Not directly (analysis only, no execution).
**Trading-AI has real execution** but only market orders via T212.
**Proposal:**
- Limit orders (T212 API supports them)
- Stop-loss / take-profit automation
- Trailing stops
- OCO (One-Cancels-Other) bracket orders

#### 14. Structured Configuration (YAML/TOML)
**DSA has:** Extensive `.env` + YAML config + LiteLLM config.
**Trading-AI gap:** Constants hardcoded in modules, only `.env` for secrets.
**Proposal:**
- `config.yaml` for model weights, risk parameters, schedule times, ticker lists
- Hot-reload on config change
- Validation with pydantic
