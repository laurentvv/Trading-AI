   - Ignorer les tests nécessitant des services externes (Ollama, T212 API, Hyperliquid, etc.)

2. **Lancer spécifiquement les tests liés aux changements récents** :
   - `tests/test_t212.py::TestSafeRequest` — 4 tests unitaires safe_request

   - `tests/test_file_locking.py` — tests atomic write et concurrent writes

### Phase 2: Analyse des échecs et diagnostic

3. **Catégoriser les échecs** :
   - Erreurs d'import (modules manquants, chemins incorrects)
   - Erreurs de logique (assertions failed)
   - Erreurs d'environnement (API keys, services externes)
   - Erreurs de compatibilité (Python 3.12+, dépendances)

4. **Vérifier les problèmes identifiés dans le code** :
   - Import `from t212_executor import get_t212_price` dans `data.py:139` — fragile car dépend du CWD et de `sys.path`
   - `test_except_fix.py` modifie le vrai fichier `STATE_FILE` — risque de corruption
   - `test_file_locking.py:9` fait `sys.path.insert(0, str(Path(__file__).parent / "src"))` — chemin incorrect (devrait être `parent.parent / "src"`)
   - `test_file_locking.py:11` importe `from t212_executor import ...` — sans le prefix `src.`, ce qui échoue avec la config pytest (`pythonpath = ["src"]`)

### Phase 3: Corrections

5. **Corriger les imports** dans `test_file_locking.py` :
   - Changer `sys.path.insert(0, str(Path(__file__).parent / "src"))` → `str(Path(__file__).parent.parent)`
   - Changer `from t212_executor import ...` → `from src.t212_executor import ...`

6. **Améliorer `test_except_fix.py`** :
   - Utiliser `tmp_path` de pytest au lieu de modifier le vrai `STATE_FILE`
   - Ou à défaut, mocker `STATE_FILE` pour pointer vers un fichier temporaire

7. **Vérifier le test `test_t212.py`** :
   - La fonction `test_connection()` ne doit pas être collectée par pytest (elle n'a pas d'assert et fait des appels réseau réels). Vérifier si pytest la ramasse et si oui, la renommer ou la protéger.

8. **Corriger tout autre échec** détecté lors de la Phase 1.

### Phase 4: Validation finale

9. **Re-lancer les tests** après corrections pour vérifier que tout passe
